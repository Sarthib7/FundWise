# Threshold-change Proposal semantics

FundWise's `threshold_change` Proposal kind currently updates only `groups.approval_threshold` (FundWise field). The Squads on-chain `threshold` — which actually governs reimbursement Proposal approval via Squads' on-chain status mapping — never changes. The two are decoupled, and the FundWise field has no effect on real money-movement governance. For Summit Berlin (2026-06-13) we **hide the `threshold_change` Proposal type** from the UI and have the server reject it with a "not yet supported" message; existing display labels for any historical row stay for back-compat. Post-Summit, we wire it to a real Squads `multisig.instructions.configTransactionCreate` so threshold changes actually update the on-chain rules. The third option (document FundWise-internal governance only) is rejected because the semantic divergence would confuse Members who see a "passed" threshold-change Proposal but reimbursements still require the old on-chain threshold. See FW-071.
