# Fund Mode take-rate monetization

Fund Mode launches with a **pool take-rate model** instead of a per-Group subscription. Free tier for everyone; Pro tier (with perks) is a later, optional surface for businesses and funds. Revenue lines for Summit Berlin (2026-06-13) are a $5 USDC Creation Fee at Treasury init, a 0.5% Contribution Fee, a 0.5% Reimbursement Fee, and a 25 bps Cross-Chain Routing Fee on CCTP/LI.FI inbound. Holding Fee (0.5%/year on AUM), Yield Fee (30% on Meteora yield), and Fiat Deposit Fee are post-Summit. All fees are operator-global (tunable via env/DB, not per-Group), and use **buyer-pays semantics** — the Treasury sees what was deposited and the Member receives what was requested, with the fee added on top of the signed transfer. Split Mode Settlements stay free per [ADR-0025](./0025-keep-split-mode-free-and-monetize-later-surfaces.md). Fees flow on-chain to `FUNDWISE_PLATFORM_FEE_WALLET` atomically inside the same transaction as the main transfer; collection is recorded in a new `platform_fee_ledger` table for reconciliation. Supersedes [ADR-0031](./0031-fund-mode-pricing-decision.md) (subscription model rejected as a consumer-acquisition barrier).
