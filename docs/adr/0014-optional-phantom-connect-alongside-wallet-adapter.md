# Optional Phantom Connect alongside wallet-adapter

Member identity stays Solana pubkey–based. The app already uses `@solana/wallet-adapter-*` for Phantom, Solflare, Backpack, and other standard wallets. We may add **Phantom Connect** (`@phantom/react-sdk`) as a second, optional path for Google/Apple sign-in and embedded wallets, configured with a Phantom Portal App ID and OAuth callback routes. **We are not** replacing the adapter or requiring Portal for the core demo: extension wallets keep working as today. Signing for Settlements and Fund Mode flows must be abstracted so both paths stay safe and do not break each other. This is harder to reverse once users rely on embedded sessions, and it is a real trade-off (extra dependency, Portal setup, dual signing surface) versus broader wallet choice without social onboarding.
