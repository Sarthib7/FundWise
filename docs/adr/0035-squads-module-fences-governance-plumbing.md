# `lib/squads/` Module fences Squads governance plumbing

All `@sqds/multisig` SDK use, PDA derivation, account decoding, status mapping, and TransactionMessage assembly for Fund Mode live behind a single fenced Module at `lib/squads/`. Public operations are high-level only — `createMultisig`, `deriveMultisigAddresses`, `proposeReimbursement` (with optional `platformFee` for FW-068's two-leg vault transactions), `review`, `execute`, `verifyTreasury`, `verifyProposalMetadata`, `verifyProposalReview`, `verifyExecution`, `loadProposal` — and callers never see `@sqds/multisig` types, PDA structure, or `TransactionMessage`/`transferInstruction` from `@solana/web3.js`. The Module owns the full wallet flow (build + sign + send + confirm) via a structural `WalletSigner` that wallet-adapter, Privy embedded wallets, and future server signers all satisfy. Every verify operation accepts an optional `AccountReader` Adapter so the Module's Interface is the unit-test surface; production callers pass nothing (defaults to a real RPC connection), tests inject mocks. The Fee Module composes with the Squads Module by passing computed `platformFee` info into `proposeReimbursement`; the Squads Module never imports Fee. Error type stays `FundWiseError` to match existing catch sites. Aligns code with [ADR-0029](./0029-squads-governance-source-of-truth-for-fund-mode.md): Squads governance is the authority layer, and now there is a Module that owns it.
