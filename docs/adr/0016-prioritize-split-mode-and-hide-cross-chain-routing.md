# Prioritize Split Mode hardening and hide LI.FI route complexity

For the hackathon story, FundWise now prioritizes Split Mode devnet readiness first, then LI.FI as the main sponsor-layer support path for EVM-first users, then Zerion and Telegram support layers, with Fund Mode proposals deferred until later. We chose this because the strongest near-term product is `Group -> Expense -> Balance -> Settlement -> Receipt`, and the most valuable sponsor addition is `Route funds for Settlement` inside the Settle flow, hiding cross-chain routing details instead of asking users to learn bridge mechanics or pre-fund from the dashboard.
