# Shipped vs Planned Product Matrix

FundWise's public story stays centered on Split Mode until mainnet launch. This matrix is the canonical reference for what can be claimed as shipped, what can be shown as demo support, and what remains planned or speculative.

FundLabs-level positioning is broader: FundLabs builds the financial layer for groups, human or AI. FundWise is the first product in that stack; Fundy and Receipt Endpoint are planned expansion products. This matrix still governs what can be claimed for the current FundWise app.

Competitive positioning: do not claim an empty category or "first crypto Splitwise." The market already includes mature web2 substitutes and several crypto-native bill-splitting attempts. FundWise should be positioned around verified USDC Settlement for private Groups, live Settlement Request Links, and Receipts.

| Surface | Current state | Public positioning |
| --- | --- | --- |
| Split Mode | Shipped devnet MVP | Primary product path: Group, Expense, Balance, Settlement, Receipt |
| Settlement Request Links | Shipped / core loop | Shareable debtor flow that resolves the live Balance, requires wallet confirmation, and never auto-sends |
| Source Currency entry | Future | Do not demo as real behavior until Exchange Rate Snapshots and converted USD/USDC ledger values are stored end to end |
| Expense Proof upload | Future | Do not demo as real behavior until storage, preview, size limits, and access rules are implemented |
| Fund Mode | Partial / invite-only, accelerated beta build | Hero product direction: Treasury and Contributions exist; Proposal lifecycle and integrations must ship before it is claimed as complete |
| Fund Mode yield / productive Treasury strategy | Planned | Directional FundLabs thesis only; do not claim live APY, yield routing, or yield spread until implemented and risk-reviewed |
| LI.FI | Support layer groundwork + readiness check | Use only as mainnet `Route funds for Settlement` when the debtor needs Solana USDC before normal Settlement; Sepolia is not a supported rehearsal path |
| Zerion | Scripted readiness demo | Read-only wallet-readiness support; not wallet connection, identity, signing, or Settlement execution |
| Agent Skill Endpoint | Shipped baseline | Public `/skill.md` and `/api/docs` discovery/reference only |
| Scoped Agent Access | Planned | Required before durable external-agent access to private Member or Group data |
| Payable Settlement Requests | Planned research | Requires Scoped Agent Access, Spending Policies, exact live amounts, expiry, idempotency, and verified payment proof before Receipt |
| Fundy | Planned separate repo | Telegram/personal-finance companion after Split Mode; read-only and draft-safe first; money movement deep-links back to the web app |
| Receipt Endpoint | Planned FundLabs product | Agent-commerce receipt API; should grow from FundWise Receipts and Payable Settlement Requests, not be claimed as live |
| Visa / IBAN / Altitude-style top-ups | Future | Non-crypto onboarding and card/bank-rail partnerships after the crypto-native core is reliable |
| Fund Mode mini-games | Out of scope for FundWise | Do not include in Split Mode, the launch story, or Fund Mode until separately justified outside prediction-market-like scope |
| Tax guidance | Future Fundy surface | Belongs to Fundy/personal-finance expansion, not the FundWise web MVP |
